# External Integrations

**Analysis Date:** 2026-05-06

## APIs & External Services

**AI Model Services:**
- MegaDetector V6 (Microsoft/CameraTraps) - Animal, person, and vehicle detection in video frames
  - SDK/Client: `megadetector` pip package; `PTDetector` from `megadetector.detection.pytorch_detector`
  - Auth: None (model downloaded directly from URL in `known_models` registry, ~160 MB, cached at `~/.cache/megadetector/`)
  - Used in: `wildlife_processor.py` — `load_megadetector()`, `run_megadetector()`

- SpeciesNet v4.0.2a (Google) - Species identification for 2,000+ species with geographic filtering
  - SDK/Client: `speciesnet` pip package; `SpeciesNet` class with `DEFAULT_MODEL = 'kaggle:google/speciesnet/pyTorch/v4.0.2a/1'`
  - Auth: Kaggle API credentials at `~/.kaggle/kaggle.json` (required for first-time model download)
  - Model cache: `~/.cache/kaggle/models/google/speciesnet/` (~640 MB, `.pkl` files)
  - Used in: `wildlife_processor.py` — `run_speciesnet()`, `--generate-taxonomy` mode, `--reprocess-flagged` mode

**Package Registry:**
- PyPI - Package version checking for dashboard update alerts
  - SDK/Client: `urllib.request` (stdlib); hits `https://pypi.org/pypi/{package}/json`
  - Auth: None
  - Used in: `web_app.py` — `api_check_updates()` endpoint (`/api/updates`)
  - Packages checked: `megadetector`, `speciesnet`

**CDN (Frontend Assets):**
- Google Fonts CDN - Typography (Playfair Display, Space Mono, Crimson Pro)
  - URL: `https://fonts.googleapis.com/css2?family=...`
  - Auth: None
  - Used in: `static/index.html`
- jsDelivr CDN - Chart.js 4.4.0 charting library
  - URL: `https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js`
  - Auth: None
  - Used in: `static/index.html`

**WordPress (Publishing):**
- Referenced in `PROJECT.md` (`wp_page_id: 215`, `wp_slug: wildlife-monitor`, `github_url`)
- No WordPress API code found in the application itself; publishing appears to be a separate workflow

## Data Storage

**Databases:**
- SQLite 3 (stdlib `sqlite3` module)
  - File path: `<data_dir>/wildlife.db` (default `./data/wildlife.db`)
  - Client: Raw `sqlite3` with `WAL` journal mode and `foreign_keys=ON`; no ORM
  - Schema managed in: `database.py` — `SCHEMA` constant + migration constants
  - Tables: `videos`, `detections`, `species`, `crops`, `blacklist`, `video_corrections`
  - Connection: `database.get_conn()` context manager; `check_same_thread=False`

**File Storage:**
- Local filesystem (temporary staging during processing)
  - Video staging: `<script_dir>/local_videos/` — populated by `nas_sync.sh`, deleted after archiving
  - Crop images: `<data_dir>/crops/` — JPEG animal bounding-box crops, permanent
  - Thumbnails: `<data_dir>/thumbnails/` — JPEG video preview stills, permanent
  - Run logs: `<data_dir>/run_YYYYMMDD_HHMMSS.log` and `<data_dir>/run_manual.log`
  - Taxonomy cache: `<data_dir>/speciesnet_classes.json` — generated by `--generate-taxonomy` flag

- NAS (Network-Attached Storage) — primary long-term video storage
  - Protocol: NFS (recommended) or SMB/CIFS
  - Mount point: configurable, default `/mnt/wildlife_nas`
  - Config: `~/.config/wildlife_monitor/nas.conf` (INI key-value, sourced by shell scripts)
  - SMB credentials: `~/.config/wildlife_monitor/nas_smb_creds` (chmod 600)
  - Archive structure on NAS: `<archive_subdir>/<camera>/<year>/<month>/<day>/<file>`
  - Blank archive: `<archive_subdir>/blanks/<camera>/<year>/<month>/<day>/<file>`
  - Managed by: `nas_sync.sh` (archiving inline Python blocks), `nas_connect.sh` (setup wizard)

**Caching:**
- MegaDetector model: `~/.cache/megadetector/*.pt` — downloaded once via `urllib.request.urlretrieve`
- SpeciesNet model: `~/.cache/kaggle/models/google/speciesnet/**/*.pkl` — downloaded via Kaggle API on first `SpeciesNet()` instantiation
- No in-memory cache or Redis equivalent

## Authentication & Identity

**Auth Provider:**
- None — no user authentication on the dashboard
- Dashboard is network-accessible with no login (`CORSMiddleware` allows all origins: `allow_origins=["*"]`)
- Security is perimeter-based (firewall/network access)

**Credentials managed externally:**
- Kaggle API key at `~/.kaggle/kaggle.json` — required for SpeciesNet model download only
- NAS SMB password stored in `~/.config/wildlife_monitor/nas_smb_creds` (chmod 600) — never exposed via API (explicitly `result.pop("NAS_SMB_PASS", None)` in `web_app.py`)

## Monitoring & Observability

**Error Tracking:**
- None (no Sentry, Rollbar, or equivalent)

**Logs:**
- Processing runs: file logs at `<data_dir>/run_YYYYMMDD_HHMMSS.log` + stdout (via `logging` stdlib)
- Manual dashboard-triggered runs: `<data_dir>/run_manual.log` (truncated on each run)
- Systemd-triggered daily runs: `<data_dir>/cron.log` (append mode)
- Dashboard process: systemd journal via `SyslogIdentifier=wildlife-monitor`
- Log format: `%(asctime)s  %(levelname)-8s  %(message)s` with `%Y-%m-%d %H:%M:%S` timestamps

**System metrics:**
- `/api/system` endpoint provides CPU %, RAM, disk usage, and process uptime via `psutil` (optional dependency)

## CI/CD & Deployment

**Hosting:**
- Self-hosted on user's Linux machine (Ubuntu 20.04+)
- Dashboard runs as a systemd service: `wildlife-monitor.service` (starts on boot, restarts on failure)
- Daily analysis runs as: `wildlife-analysis.service` triggered by `wildlife-analysis.timer` at 06:00

**Systemd units:**
- `systemd/wildlife-monitor.service` — dashboard service (port 8080, host 0.0.0.0)
- `systemd/wildlife-analysis.service` — oneshot processor service (4h timeout)
- `systemd/wildlife-analysis.timer` — daily calendar timer (6 AM, persistent, 60s random delay)

**CI Pipeline:**
- None (no GitHub Actions, CircleCI, or equivalent detected)

**Version control:**
- GitHub: `https://github.com/twostar01/wildlife-monitor`
- Version tracked in `VERSION` file (currently `1.0.0`)

## Environment Configuration

**Required env/config:**
- `~/.kaggle/kaggle.json` — Kaggle API credentials (`{"username": "...", "key": "..."}`) for SpeciesNet download
- `~/.config/wildlife_monitor/nas.conf` — NAS connection config (set up by `nas_connect.sh`)
  - Keys: `NAS_HOST`, `NAS_MOUNT`, `NAS_SHARE`, `NAS_PROTOCOL` (nfs|smb), `NAS_VIDEO_SUBDIR`, `NAS_ARCHIVE_SUBDIR`
- `<data_dir>/settings.json` — Processing settings (managed by dashboard UI; falls back to defaults if absent)

**Optional config:**
- `wildlife.conf` — INI file alongside `web_app.py` for port/host/data_dir overrides

**Secrets location:**
- Kaggle key: `~/.kaggle/kaggle.json` (standard Kaggle location)
- NAS SMB password: `~/.config/wildlife_monitor/nas_smb_creds` (chmod 600, never logged or returned by API)

## Webhooks & Callbacks

**Incoming:**
- None — no webhook receivers

**Outgoing:**
- None — no webhook calls; the dashboard-triggered run (`/api/run`) spawns a local subprocess (`nas_sync.sh`) rather than calling external services
